## üá¨üáß English Version

### 1. Project Summary: Whac-A-Mole
We created a digital version of the classic arcade game "Whac-A-Mole." It is a single-page website where cartoon moles pop out of holes, and you click them to score points before the 30-second timer runs out.

### 2. How to Run the Game
1.  **Start the Server:**
    ```bash
    nohup python3 -m http.server 3000 > /dev/null 2>&1 &
    ```
    *(Note: We use `nohup` to keep the server running in cloud environments)*
2.  **Play:** Open your browser to port 3000 (e.g., `http://localhost:3000`).

### 3. How it Works (The "Ingredients")
Think of this website like a house. We used three main tools:
1.  **HTML (The Skeleton):** Builds the structure (title, button, grid of 9 holes).
2.  **CSS (The Decorator):** Handles colors, fonts, and layout.
    *   **Special Feature:** Instead of image files (`.jpg`), we used **SVG Data URIs** to "draw" the moles with code. This ensures zero external dependencies and instant loading.
3.  **JavaScript (The Brains):** Handles the game logic (randomizing moles, timer, scoring).

### 4. Lessons Learned
*   **"Drawing with Code" vs. Images:** Using CSS/SVG instead of image files makes the project self-contained. It's like memorizing a drawing instead of carrying a photo‚Äîyou can recreate it anywhere.
*   **Cloud Server Persistence:** In cloud IDEs, standard background processes often die.
    *   **The Fix:** Use `nohup`. This acts like a "Do Not Disturb" sign, keeping the server alive even when the shell is idle.
*   **Game Loop Cleanup:** We learned to explicitly `clearInterval` when restarting the game to prevent "zombie" timers.

### Development Workflows

#### Creating Issues
When starting a new feature or bug fix:

```bash
# 1. Update main branch
git checkout main && git pull

# 2. Create a detailed issue
gh issue create --title "feat: Descriptive title" --body "$(cat <<'EOF'
## Overview
Brief description of the feature/bug.

## Current State
What exists now.

## Proposed Solution
What should be implemented.
EOF
)"
```

#### Standard Development Flow
1.  **Create Branch:** `git checkout -b feat/issue-number-description`
2.  **Make Changes:** Implement features.
3.  **Test:** Verify changes.
4.  **Commit:** `git commit -m "feat: description ... Closes #issue"`
5.  **Push:** `git push -u origin branch-name`
6.  **Create PR:** `gh pr create`
7.  **Wait:** Do not merge yourself.
